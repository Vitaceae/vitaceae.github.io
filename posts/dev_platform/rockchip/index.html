## Build Images

### Get SDK 

$ sudo apt update
$ sudo apt install -y repo git python

$ WORK_DIR=~/work/firefly_px30
$ mkdir -p $WORK_DIR; cd $WORK_DIR
# full SDK: px30_linux_release.xml
# BSP(basic repositories and compile tools): px30_linux_bsp_release.xml
$ repo init \
    --repo-url https://gitlab.com/firefly-linux/git-repo.git \
    -u https://gitlab.com/firefly-linux/manifests.git \
    -b master \
    -m px30_linux_release.xml

# sync
$ .repo/repo/repo sync -c --no-tags
$ .repo/repo/repo start firefly --all

# update
$ .repo/repo/repo sync -c --no-tags

Build commands

# Select the board configuration
#   px30-ubuntu.mk: Ubuntu, MIPI
#   px30-lvds-ubuntu.mk: Ubuntu, LVDS
#   (the soft link points to device/rockchip/px30/*.mk)
$ ./build.sh px30-lvds-ubuntu.mk

# Build target: all | kernel | uboot | recovery | updateimg 
$ ./build.sh all

# confirm the links in rockdev/ and tools/linux/Linux_Pack_Firmware/rockdev/package-file
$ ./mkfirmware.sh

# clean build
$ ./build.sh cleanall

## Storage Partitions

uboot uboot.img

boot boot.img(kernel image; device tree)

## Image Tools

### Windows

  1. Install DriverAssitant
  2. Connect to the device via USB cable
  3. Hold RECOVERY button and then power on; release RECOVERY button after 2 sec.
  4. Run AndroidTool.exe

### Linux 

$ cd Linux_Upgrade_Tool
$ sudo mv upgrade_tool /usr/local/bin
$ sudo chown root:root /usr/local/bin/upgrade_tool
$ sudo chmod a+x /usr/local/bin/upgrade_tool

## Boot Flow

系統初始化

  1. power-on initialization
  2. run BootRom code on SRAM
  3. verify bootloader
  4. load and run bootloader
    • initialize DDR SDRAM
    • shadow bootloader to DDR memory
    • load and run Linux kernel

Boot mode

  • Normal
  • Loader
    • connect USB cable to device, hold recovery key and then power-on
    • to upgrade firmware
      • full image: update.img
      • partition images: kernel.img, rootfs.img, recovery.img…
  • MaskRom + used when bootloader is corrupted

## Custom Changes

### Change Board config

在 device/Rockchip/rk3326/Boardconfig.mk 中確認指定的 dts

#export RK_KERNEL_DTS=rk3326-evb-lp3-v10-linux
#export RK_KERNEL_DTS=rk3326-evb-ai-va-v11-linux

使用腳本修改 board type, 間接更改 soft link 對象

$ ./build.sh px30-lvds-ubuntu.mk

修改 Ubuntu 分割區配置: device/rockchip/px30/parameter-ubuntu.txt

-CMDLINE: mtdparts=rk29xxnand:0x00002000@0x00004000(uboot),0x00002000@0x00006000(trust),0x00002000@0x00008000(misc),0x00010000@0x0000a000(boot),0x00010000@0x0001a000(recovery),0x00010000@0x0002a000(backup),0x00020000@0x0003a000(oem),0x00c00000@0x0005a000(rootfs),-@0x00c5a000(userdata:grow)

## Problems

E: Unable to locate package repo

$ mkdir ~/bin
$ curl https://storage.googleapis.com/git-repo-downloads/repo > ~/bin/repo
$ chmod a+x ~/bin/repo
$ echo PATH=~/bin:$PATH >> ~/.bashrc

rockchip-vop ff470000.vop: [drm:vop_isr] ERROR WIN1_EMPTY irq err

bandwidth not enough

  1. disable dmc, 關閉 DDR 變頻功能
&dmc {
    center-supply = <&vdd_logic>;
    status = "disabled";
};

若問題消失表示 DDR 頻率正確, 開啟 dmc 並調整 vop-dclk-mode 值: 0 | 1 | 2

vop-dclk-mode = <0>;
  1. 若問題仍然存在表示 DDR 頻率異常, 須調整成最大支援頻率
/*system status freq(KHz)*/
SYS_STATUS_NORMAL 856000

dmc 參數說明(.dts)

  • auto-freq-en=1 時
    • upthreshold: 超過指定值(使用率)時升頻
    • downdifferential: 低於指定值(使用率)時降頻
  • auto-freq-en=0 時
    • 除特殊條件外, 使用 system-status-freq 中的 SYS_STATUS_NORMAL(kHz)
