# Installation

## Windows

  • install chocolatey 
    > Set-ExecutionPolicy Bypass -Scope Process -Force; [System.Net.ServicePointManager]::SecurityProtocol = [System.Net.ServicePointManager]::SecurityProtocol -bor 3072; iex ((New-Object System.Net.WebClient).DownloadString('https://community.chocolatey.org/install.ps1'))
  • install Go stable version: to build Hugo from source and use the Hugo Modules feature
    • from binary
    • from choco command 
      > choco install golang sass hugo-extended
  • install Dart Sass: to transpile Sass to CSS
  • install Hugo
    • from prebuilt-binary
      • add this directory to PATH variable
    • from choco command

## Linux 

$ sudo snap install hugo

# enable/revoke access to removable media
$ sudo snap connect hugo:removable-media
$ sudo snap disconnect hugo:removable-media

# enable/revoke access to SSH keys
$ sudo snap connect hugo:ssh-keys
$ sudo snap disconnect hugo:ssh-keys

# Directory structure

  • Hugo: Content organization
.
├── archetypes  # templates for new content
├── assets      # pipeline which global resources typically passed through
│                 (images, CSS, Sass, JavaScript, and TypeScript...)
├── config      # (optional)site configuration
│   ├── _default
│   │   ├── hugo.toml
│   │   ├── languages.toml
│   │   ├── menus.en.toml
│   │   └── params.toml
│   ├── production
│   │   ├── hugo.toml
│   │   └── params.toml
│   └── staging
│       └── ...
├── content     # markup files (typically markdown)
├── data        # JSON, TOML, YAML, or XML; accessible as a map in the .Site.Data variable
├── layouts     # templates to transform content, data, and resources into a complete website
├── resources   # cached output from Hugo's asset pipelines
├── static      # files that will be copied to the public directory
├── themes
└── hugo.toml

# Quick Start

  • Hugo: Basic usage
  • Windows user: run commands from PowerShell(not Windows PowerShell), WSL or Git Bash

## Create a demo site on WSL 

$ SITE_NAME=hugo_site
$ hugo new site $SITE_NAME
$ cd $SITE_NAME
$ git init

# add a new theme
$ git submodule add https://github.com/theNewDynamic/gohugo-theme-ananke.git themes/ananke
$ echo "theme = 'ananke'" >> hugo.toml

# write a new post; set 'draft = false' in the .md file when done
$ hugo new posts/new_post.md

# need to find the WSL's LAN IP; add an alias function in bash.rc
function hugo_server() {
    WSL_IP=$(ip addr show eth0 | grep "inet\b" | awk '{print $2}' | cut -d/ -f1)
    hugo server --bind $WSL_IP --baseURL=http://$WSL_IP -D -F --gc -w
}

# run local web server(changes take effect on the fly)
$ hugo_server

## Host on GitHub Pages

Publish to GitHub Pages (there’s rendering problem following the official method)

  1. add .github/workflows/hugo.yaml
  2. create a repo named github_user_name.github.io
  3. run hugo and push the contents in public to the repo
  4. GitHub repo::Settings::Pages::Github Actions::Deploy from a branch
  5. run https://github_user_name.github.io in web browser
